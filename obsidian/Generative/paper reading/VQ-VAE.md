
Vector Quantized Variational AutoEncoder
이미지, 음성, 텍스트 같은 연속적인 데이터를 "정수 토큰의 배열"로 압축하는 오토 인코더

## 배경

예를 들어 픽셀 크기 256의 정사각형 이미지라고 가정하면 일반적인 오토 인코더 모델은 다음과 같이 동작할 수 있다. 인코더는 정보를 압축하고 디코더는 복원한다.
```
입력 이미지: [H = 256, W = 256, C = 3]
Encoder 출력: [h = 16, w = 16, d = 256]
Decoder 출력: [H = 256, W = 256, C = 3]
```

문제는 오토 인코더가 출력하는 값이 보통 연속적인 실수 벡터라는 점이다. 예를 들어 다음과 같을 수 있다.
```
z[0,0] = [0.13, -1.52, 0.44, ..., 0.91]
z[0,1] = [0.20, -1.48, 0.39, ..., 0.88]
...
```

VQ-VAE는 오토 인코더가 만든 연속 실수 벡터를 그대로 사용하지 않고 미리 준비된 codebook이라는 벡터 사전에서 가장 가까운 벡터로 바꿔서 쓴다. 즉, VQ-VAE 모델이 동작 흐름을 간략히 서술하면 다음과 같다.
```
입력 이미지
→ Encoder
→ 연속 latent 벡터들
→ 가장 가까운 codebook vector로 치환
→ Decoder
→ 복원 이미지
```

배경 단락에서 서술한 예시와 같이 인코더가 내놓는 latent가 \[16, 16, 256] 크기의 텐서라면 총 256개의 latent vector가 생성된 것이고 각 벡터는 256차원이다. 

Codebook은 학습 가능한 벡터들의 사전이고 각 벡터 차원은 latent 벡터와 같은 256차원이다. 만약 codebook size가 1024라고 가정하면 \[1024, 256] 크기의 텐서이고 256개의 각 latent 벡터는 1024개의 사전 벡터들 중 가장 가까운 것으로 치환된다.

즉, 인코더 출력(연속 실수 벡터 \[16, 16, 256])을 입력으로 받은 VQ(Vector Quantization) 단계에서 \[16, 16] 크기의 codebook 정수 인덱스(codebook 안 몇 번째 벡터로 치환되었는지) 맵 텐서가 나온다. 디코더 입력을 위해서는 정수 인덱스 텐서를 \[16, 16, 256] 텐서로 변환해야 한다. 디코더 입력으로는 codebook vector 만이 들어간다. 디코더는 인코더의 출력을 직접 볼 수 없다.

직관적으로는 몇 개의 codebook 벡터만으로 복잡한 데이터를 원복할 수 있을까? 라는 의문이 들지만 위 예시의 경우 총 256개의 codebook 벡터가 이미지 하나에 대응될 수 있다. codebook size가 1024이므로 총 $1024^{256}$개 경우의 수가 존재할 수 있어 표현력은 충분하다.
