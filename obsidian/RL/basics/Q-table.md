
[[Value Function]]에서 등장하는 Q-value를 값으로 갖는 테이블. 가능한 모든 상태, 모든 행동들에 대해 정리된 테이블을 의미한다.

예를 들어 S1, S2, S3라는 상태 3개, 각 상태에서 가능한 행동은 Left, Right로 두 가지가 존재하는 게임이 있다고 가정하면 Q-table은 다음과 같을 수 있다.

|     | Left | Right |
| --- | ---- | ----- |
| S1  | 0.1  | 0.8   |
| S2  | -0.5 | 0.5   |
| S3  | 0.3  | 0.2   |
에이전트는 현재 상태의 행에서 가장 높은 Q-value를 갖는 행동을 선택하여 행동한다.
$$
a^* = \arg\max_a Q(s, a)
$$
Q-table의 값을 처음부터 알고 있을 수는 없기에 에이전트는 직접 행동해보고 보상을 받으며 테이블의 값을 업데이트한다. 업데이트 과정은 [[Q-Learning]]을 활용한다.

## Q-table의 단점

상황이 복잡해질수록 표의 크기가 너무 커진다. 혹은 센서값처럼 연속적인 값이 들어오는 경우 이를 표로 표현할 수 없다.
