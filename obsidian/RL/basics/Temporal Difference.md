
Temporal Difference Error, TD error라고도 부른다.

현재 시점의 추정값과 다음 시점 정보를 반영한 추정값의 차이를 본다.

TD error는 현재 추정값이 Bellman Equation을 얼마나 어기고 있는지 측정하는 값이라고도 할 수 있다. 예를 들어 TD error가 0이면 Bellman Equation이 완벽하게 성립하는 것이다. TD 학습은 Bellman Equation을 맞추는 과정이라고도 볼 수 있다.

# Basic TD error formula
# $$
\delta_t = R_{t+1} + \gamma V(S_{t+1}) - V(S_t)
$$
### $R_{t+1}$

현재 상태에서 행동한 직후 받는 즉시 보상

### $\gamma V(S_{t+1})$

다음 상태 $S_{t+1}$의 가치를 할인한 값. 미래 가치의 대리 표현

### $V(S_t)$

현재 상태 $S_t$에 대한 기존 추정값.

내가 원래 믿고 있던 값이자 업데이트해야할 대상.

### $R_{t+1} + \gamma V(S_{t+1})$

현재 상태 가치에 대한 목표값.

TD target이라고 부른다.

- ## TD error를 활용한 업데이트

TD error가 양수 -> 목표값이 기존 추정보다 더 크다. -> 기존 추정 값을 더 키워야 한다.
TD error가 음수 -> 목표값이 기존 추정보다 더 작다. -> 기존 추정 값을 더 줄여야 한다.
# $$
V(S_t) \leftarrow V(S_t) + \alpha \delta_t
$$
$\alpha$는 학습률

오차가 양수면 값을 올리고, 오차가 음수면 값을 내리고, 오차의 크기만큼 값을 조정

TD error는 학습 신호

# Monte Carlo와의 차이점

몬테카를로는 에피소드가 끝날 때까지 기다린 다음 실제 $return$
$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} \, ...$ 을 가지고 학습한다.
즉, 실제로 먼 미래 보상까지 다 확인한 다음 업데이트한다.

반면 TD는 에피소드 끝까지 기다리지 않고 한 스텝만 본 뒤 $R_{t+1} + \gamma V(S_{t+1})$ 를 타겟으로 삼는다. 미래 상태 대신 다음 상태의 추정값을 사용하는 이 방식을 bootstrapping이라고 한다.

따라서 TD error는 "정답과의 차이"가 아니라 "한 스텝 bootstrap target과의 차이"이다.

# TD의 장단점

장점
- 에피소드 끝까지 기다릴 필요가 없다.
- 온라인 학습이 가능하다.
- 긴 문제에서도 빠르게 업데이트가 가능하다.
- 몬테카를로에 비교하면 학습이 빠르고 안정적이다. (구체적인 이유는 단점과 함께 후술)

단점
- 추정값으로 또 추정값을 가르치므로 편향이 생길 수 있다.
	- 몬테카를로는 실제 return을 끝까지 보기 때문에 편향은 작다. 하지만 실제로 나오는 에피소드의 결과가 운 등 여러 요소에 의해 흔들리는 경우가 많아 분산이 크다.
	- TD는 추정값을 타겟으로 삼는 과정에서 추정값 안의 노이즈, 오차가 다시 타겟이 되며 평균적으로 치우친 값을 학습할 수 있다. 다만 미래 전체 과정의 랜덤성을 현재의 추정치로 단순화하며 분산을 줄일 수 있다.
		- 미래 과정의 랜덤성을 현재 추정치로 단순화한다는 의미
		- 원래는 $G_t = R_{t+1} + \gamma G_{t+1}$ 형태의 재귀식(매 순간의 즉시 보상 $R_{t+n}$ 들이 개별적으로 환경의 랜덤성을 지닌다.)인데 TD에서는 이 식을 $G_t \approx R_{t+1} + \gamma V(S_{t+1})$ 이라고 근사해버려 환경의 랜덤성이 직접적으로 반영되지 않아 분산이 작다.
