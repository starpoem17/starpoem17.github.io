# 시스템 구조와 재설계 방향

## 1. 전체 구조

이 프로젝트는 아래 네 층으로 이해하면 된다.

- `personal/`: 상위 기준 문서
- `plans/`: 구현과 운영 계획
- `obsidian/`: 사용자가 직접 관리하는 원본 노트
- Quartz 설정/생성 코드: 실제 공개 사이트를 만드는 코드

이번 재설계의 핵심은, 이 구조를 유지한 채 공개 표면만 reference 중심으로 다시 짜는 것이다.

## 2. 입력과 출력

### 입력

- `obsidian/` 아래의 Markdown 노트
- `obsidian/` 아래 첨부 이미지
- 사이트용 프로필 이미지

입력 구조는 항상 `obsidian/` 전체를 기준으로 설명해야 한다.
현재 실제 콘텐츠는 `obsidian/Datascience` 중심이지만, 이후 아래처럼 확장될 수 있다.

```text
obsidian
  Datascience
    RL
      Actor-Critic
        A2C.md
  CUDA
    FlashAttention.md
```

### 출력

- 홈 페이지
- 최상위 섹션 페이지
- 중간 폴더 페이지
- 개별 노트 페이지
- 정적 자산

공개 URL은 입력용 `obsidian` 접두사를 드러내지 않고 섹션/폴더 구조만 반영해야 한다.

## 3. 홈 페이지 설계 원칙

홈은 랜딩 카드가 아니라 reference형 문서 소개 페이지로 재구성한다.

- 상단 `About / Notes` 버튼은 두지 않는다
- `About Me` 제목부터 시작한다
- 프로필 사진과 소개는 카드가 아니라 본문 흐름 안에 둔다
- 별도 `Notes` 단락은 두지 않는다
- 소개 아래에는 연구 관심사 라벨 2개만 둔다

즉, 홈은 “큰 카드 하나”가 아니라 “소개 섹션 + 관심사 라벨” 구조여야 한다.

## 4. 왼쪽 Explorer 설계 원칙

Explorer는 Quartz 기본 동작에 최소 설정만 얹는 수준이 아니라, reference UX를 만족하는 쪽으로 다시 본다.

필수 조건은 다음과 같다.

- 헤더는 `Notes`
- `Notes`를 누르면 실제로 열리고 닫히는 반응이 있어야 한다
- 다층 폴더 구조를 유지해야 한다
- 깊은 구조에서도 시각적으로 이해 가능해야 한다
- `Dark mode`는 좌측 패널에 두지 않는다

현재 문제는 “토글 로직은 있으나 접힘 상태가 시각적으로 충분히 드러나지 않는 것”으로 본다.
따라서 이번 설계에서는 토글 자체와 토글의 시각 반응을 같이 다룬다.

## 5. 콘텐츠 생성 파이프라인 원칙

`scripts/generate-content.ts`는 계속 아래 역할을 가진다.

- `obsidian/` 전체 탐색
- Markdown과 자산 수집
- Quartz용 `content/` 생성
- 섹션/폴더/노트 인덱스 생성
- 내부 링크 변환

하지만 홈 템플릿 생성은 reference 기준으로 다시 정리해야 한다.

- literal anchor 문법을 남기지 않는다
- 카드형 마크업을 생성하지 않는다
- `About Me`와 `Notes`가 자연스럽게 이어지게 만든다

## 6. 스타일/레이아웃 재설계 원칙

스타일은 현재의 “부드러운 카드형 랜딩”보다 reference의 “단정한 문서형 소개”에 가깝게 조정한다.

- 과한 카드 배경 제거
- 본문형 여백과 구분선 중심
- 버튼은 간결한 pill 형태 유지 가능
- 연구 관심사 라벨은 reference의 강조 박스 위치와 흐름을 따름
- 탐색기 접힘/펼침 상태가 명확히 보이게 조정
- 글꼴은 reference 쪽 타이포그래피에 맞춤

즉, 이번 작업에서 스타일은 장식보다 구조와 사용성을 우선한다.
