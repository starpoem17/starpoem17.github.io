# Quartz + Obsidian 프로젝트 개요

## 1. 이 프로젝트는 무엇인가

이 프로젝트는 `Obsidian`에 정리해 둔 Markdown 노트를 `Quartz`를 이용해 정적 웹사이트로 공개하는 작업이다.

핵심 목표는 아래와 같다.

- 원본 노트는 그대로 보존한다.
- 웹사이트에 맞는 형식은 별도로 생성한다.
- `GitHub Pages`에 올릴 수 있는 정적 사이트를 만든다.
- 처음 보는 사람도 구조를 이해하고 유지보수할 수 있게 만든다.

현재 이 프로젝트는 `Datascience` 주제의 노트들을 대상으로 구성되어 있다.

- 카테고리: `RL`, `RNN`, `Transformer`
- 원본 위치: `obsidian/Datascience`
- 사이트용 생성 위치: `content/`

즉, 이 저장소는 “원본 노트 보관소”이면서 동시에 “Quartz 사이트 프로젝트”이다.

## 2. 왜 원본을 직접 수정하지 않는가

이번 구현의 중요한 원칙은 `obsidian/` 아래 원본 `.md` 파일을 수정하지 않는 것이다.

이렇게 한 이유는 다음과 같다.

- Obsidian에서 쓰던 원본 노트를 안전하게 유지할 수 있다.
- Quartz용 변환 규칙을 바꿔도 원본은 손상되지 않는다.
- 나중에 다른 사이트 생성기로 옮길 때도 원본 자산을 그대로 재사용할 수 있다.

그래서 이 프로젝트는 아래 두 층으로 나뉜다.

1. 원본 층: `obsidian/Datascience`
2. 공개용 층: `content/`

중간에서 두 층을 연결하는 것이 변환 스크립트다.

## 3. 최종적으로 사용자가 보게 되는 사이트

사이트는 일반적인 블로그보다 `개인 소개 + 기술 노트 아카이브`에 더 가깝다.

주요 화면 구성은 다음과 같다.

- 홈 화면 중앙: `Hwajoong Kim` 소개
- 홈 화면 링크: GitHub, LinkedIn, Email
- 사이드바: 카테고리별 노트 탐색
- 카테고리 페이지: 간단한 설명 + 정렬된 노트 목록
- 노트 페이지: 원본 내용을 최대한 유지한 기술 노트

즉, 방문자는 먼저 작성자가 누구인지 보고, 이후 카테고리별로 정리된 노트를 탐색하게 된다.

## 4. 현재 구현의 핵심 원칙

이 프로젝트 구현은 아래 원칙을 기준으로 만들어졌다.

### 원칙 1. 수정 가능한 값은 한 곳에 모은다

사이트 제목, 소개 문구, 링크, 카테고리 설명, 노트 순서처럼 나중에 바뀔 수 있는 값은 한 파일 상단에 모아두었다.

해당 파일:

- `site.config.ts`

이 구조 덕분에 사용자는 복잡한 코드 내부를 뒤지지 않고도 주요 정보를 수정할 수 있다.

### 원칙 2. 자동 생성은 스크립트로 처리한다

노트 복사, frontmatter 추가, Obsidian 링크 변환, 이미지 경로 처리 같은 반복 작업은 수동으로 하지 않고 스크립트로 자동화했다.

해당 파일:

- `scripts/generate-content.ts`

### 원칙 3. Quartz 기본 구조는 유지하되 필요한 부분만 커스터마이징한다

Quartz 전체를 새로 만드는 대신, 공식 구조를 그대로 받아와 설정과 레이아웃만 프로젝트 목적에 맞게 조정했다.

해당 파일:

- `quartz.config.ts`
- `quartz.layout.ts`
- `quartz.custom.ts`

## 5. 이 프로젝트를 이해할 때 가장 먼저 봐야 할 파일

처음 보는 사람이면 아래 순서로 읽는 것이 가장 빠르다.

1. `site.config.ts`
2. `scripts/generate-content.ts`
3. `quartz.config.ts`
4. `quartz.layout.ts`
5. `.github/workflows/deploy.yml`

이 순서로 보면 “무엇을 보여줄지”, “어떻게 생성하는지”, “사이트가 어떻게 렌더링되는지”, “어떻게 배포되는지”가 자연스럽게 연결된다.

## 6. 이 프로젝트의 현재 완료 범위

현재 구현된 범위는 다음과 같다.

- Quartz 프로젝트 스캐폴드 반영
- Obsidian 원본 노트 14개를 `content/`로 자동 생성
- RNN 이미지 자산 복사
- 홈 소개 페이지 구성
- 카테고리별 인덱스 페이지 생성
- 노트 정렬 규칙 적용
- GitHub Pages용 GitHub Actions 워크플로 추가

아직 사용자가 직접 해야 하는 일은 아래 정도다.

- GitHub 저장소에 푸시
- GitHub Pages에서 `GitHub Actions`를 소스로 선택
- 필요하면 소개글과 링크 문구 수정
